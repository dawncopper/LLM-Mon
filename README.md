# LLM 服务质量实时监控平台

通过 AI 实时监测大模型服务质量的网页监控平台，通过固定周期收集影响质量的指标数据来实现感知服务质量。不仅监控可用性，更能检测模型降级、算力缩水和服务不透明问题。

## 功能特性

### 核心监控

- **实时质量监控**：固定周期采样，实时展示响应时间、成功率、错误率
- **多模型管理**：支持添加多个 LLM 模型，卡片式展示
- **API Key 管理**：支持 OpenAI、Anthropic、Azure 等多家服务商
- **繁忙度可视化**：彩色圆角矩形指示器，直观展示服务繁忙程度
- **趋势图表**：迷你折线图展示响应时间历史趋势
- **本地存储**：配置数据保存在浏览器 LocalStorage
- **响应式设计**：适配桌面、平板、移动端

### 高级质量检测

- **多维度测试用例**：内置基础响应、推理测试、代码测试等多种测试类型
- **Juice 指纹检测**：自动提取模型推理深度参数，检测后台路由切换
- **质量分数**：综合成功率、一致性、速度的加权评分（0-100）
- **一致性检测**：基于响应时间变异系数（CV）计算输出稳定性
- **测试明细**：可展开查看每个测试用例的详细结果和输出片段
- **自定义测试提示词**：在设置页面添加、编辑、删除测试用例

### 全局概览与告警

- **健康概览**：Dashboard 顶部展示整体状态、平均质量分、平均响应时间、平均成功率
- **状态分类**：健康（≥80分）/ 警告（50-79分）/ 异常（<50分）
- **智能告警**：自动检测质量分过低、成功率下降、Juice 值偏低等问题
- **告警面板**：集中展示所有异常模型及具体原因

## 使用说明

### 1. 配置 API Key

首次使用时，系统会自动跳转到设置页面。点击「添加」按钮，输入：

- 名称：便于识别的标签
- 服务商：OpenAI / Anthropic / Azure / 自定义
- API Key：对应的 API 密钥

### 2. 添加监控模型

在监控面板点击「添加模型」，填写：

- 模型名称：如 GPT-4、Claude 3 等
- API 文档地址（可选）：模型 API 文档链接
- API 端点：API 请求地址
- 选择 API Key：使用已配置的 API Key

### 3. 查看质量指标

每张模型卡片展示：

| 指标 | 说明 |
|------|------|
| 响应时间 | 最近采样的平均响应时间（ms） |
| 成功率 | 成功请求 / 总请求 × 100% |
| 质量分 | 综合评分（成功率 40% + 一致性 30% + 速度 30%） |
| Juice | 模型推理深度指纹（检测路由切换） |
| 一致性 | 输出稳定性评分 |
| 错误率 | 失败请求 / 总请求 × 100% |
| 繁忙度 | 空闲/正常/繁忙/危险 四档 |

点击「查看测试明细」可展开查看每个测试用例的详细结果。

### 4. 自定义测试用例

在设置页面的「测试用例」模块可以：

- 查看内置测试用例（基础响应、Juice 指纹、推理测试、代码测试）
- 添加自定义测试用例
- 编辑已有测试用例
- 删除不需要的测试用例

测试用例支持以下分类：
- **基础**：简单的连通性测试
- **推理**：复杂逻辑推理能力测试
- **指纹**：提取模型内部参数（如 Juice 值）
- **自定义**：用户自定义测试

### 5. 采样间隔设置

在设置页面可配置采样周期：15秒 / 30秒 / 60秒 / 120秒

## 繁忙度等级

| 等级 | 响应时间范围 | 颜色 | 圆角 |
|------|-------------|------|------|
| 空闲 | < 500ms | 薄荷绿 | 4px |
| 正常 | 500ms - 2000ms | 天际蓝 | 4px |
| 繁忙 | 2000ms - 5000ms | 琥珀橙 | 8px |
| 危险 | > 5000ms | 错误红 | 12px |

## 质量评分算法

### 质量分（Quality Score）

综合评估模型服务质量，范围 0-100：

```
质量分 = 成功率 × 0.4 + 一致性 × 0.3 + 速度分 × 0.3
```

- **成功率**：成功请求 / 总请求 × 100
- **一致性**：基于响应时间变异系数（CV），越稳定得分越高
- **速度分**：<1000ms=100, <3000ms=75, <5000ms=50, ≥5000ms=25

### Juice 指纹检测

通过特定提示词让模型输出内部推理深度参数，用于检测：
- 后台模型路由切换（如高算力版被切换到低算力版）
- 算力缩水/降级
- 服务不透明问题

参考值：
- 高算力：≥ 512
- 中算力：256 - 511
- 低算力：128 - 255
- 极低：< 128

## 告警规则

系统自动检测以下异常并触发告警：

| 类型 | 触发条件 | 级别 |
|------|---------|------|
| 质量分过低 | 质量分 < 50 | 严重 |
| 成功率下降 | 成功率 < 80% | 警告 |
| Juice 值偏低 | Juice < 256 | 警告 |

## 数据说明

- 历史数据：保留最近 100 条采样记录
- 测试结果：每个测试用例保留最近 20 条结果
- 响应时间：滑动窗口平均值
- 成功率/错误率：基于历史数据计算
- 数据存储：全部保存在浏览器 LocalStorage

## 部署

### GitHub Pages

项目已配置 GitHub Actions 自动部署工作流（`.github/workflows/deploy.yml`），推送到 `main` 分支后自动构建并部署到 GitHub Pages。

在仓库设置中启用 Pages：
1. 进入 Settings → Pages
2. Source 选择 **GitHub Actions**
3. 保存后等待自动部署完成
